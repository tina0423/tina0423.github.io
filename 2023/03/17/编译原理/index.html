自动机（Automaton）

元胞自动机，生命游戏

NFA

  • 定义：非确定性有穷自动机，由五个要素构成

    • 字母表Σ

    • 有穷的状态集合S

    • 唯一的初始状态s0 ∈ S

    • 状态转移函数 δ

      1在b上有定义，为了在a上也有定义以符合函数的特性，约定所有没有对应出边的字符指向一个不存在的空状态

      在当前状态上看到某一字母，会将状态转移到另一个幂集上

    • 接收状态集合 F 包含于 S

​ 非确定性的体现：

当看到a后可能不转移状态，也有可能转移到1状态下；1状态下也可以自发地转移到3状态（前提是有1->3的线）

  • 作用：可以识别（接受/拒绝）Σ上的字符串

  • 接受:

    （非确定性）有穷自动机A 接受字符串x,当且仅当存在一条从开始状态s0到某个接受状态（上图中为3）f ∈F、标号为x的路径

DFA

包含字母表，有穷状态集合，唯一的初始状态，状态转移函数（S x Σ -> S）接受状态集合

没有对应出边的字符默认指向一个”死状态“，死状态不管获取什么字符都会到自身

可以看到NFA简洁易于理解，便于描述语言L(A)，DFA易于判断x属于L(A)，适合产生词法分析器

数组引用的中间代码翻译

  • L可以是数组

  • 声明：int a[2] [3]

  • 数组引用：x = a[1] [2] ; a[1] [2] = x

  • 需要计算a[1] [2]相对于数组基地址a的偏移地址

    addr(a[1] [2]) = base + 1 * 12 + 2 * 4

​

  • 方法：在每次返回时传递类型宽度

  • 说明：从右树的最左开始，当到达i时，已知每一行经过的偏移量为12（3 * 4），因此总偏移量为i*12，并存入t1中，接着往上走到达j，而a[i] [j]的偏移量为4，所以t2 = j * 4,最后t3 = 总偏移 = t1 + t2， t4为a的首地址+t3偏移地址相加，并寻找内容，t5是最后的求和

  • 放入llvm中得到的代码和上面理论分析得到的不一致，主要是因为getelementptr函数

    • %2 = alloca [2 x [3 x i32]], align 16

      （%2的类型为[2 x [3 x i32]]*）

    • getelementptr: < base-type > ,< base- type >* < base-addr >, [i32 < index> ]+

      • < base-type >:不会改变指针类型，而是去根据basetype大小 去做指针的偏移（一般用不上，a++可能会出错）
      • 逗号后的几个[i32 < index> ]+才是决定访问数组中的哪个元素
      • 先找a[i] 的位置，放入%11，再找a[i] [j]的位置(< base-addr>为%11）放入%14（这个是指针），再把结果读入到%15中
